Doing Stuff with Web Things

Local Storage

  • Information stored or shared with the user is called output. Anything coming in from where it was stored or anything the user enters is called input.
  • Works in key-value pairs
  • Stored locally
  • Information can be used multiple times and called over multiple sessions
<!doctype html>
<html>
<head>
    <title>Story Editor</title>
    <script>
        function loadStory(){
            var storyName = document.getElementById("name_input").value
            var storyHTML = localStorage.getItem(storyName)
            document.getElementById("story_editor").value = storyHTML
        }
        function saveStory(){
            var storyName = document.getElementById("name_input").value
            var storyHTML = document.getElementById("story_editor").value
            localStorage.setItem(storyName,storyHTML)
        }
        function displayStory(){
            var storyHTML = document.getElementById("story_editor").value
            document.getElementById("story_display").innerHTML = storyHTML
        }
    </script>
                
    <style>
        textarea{
            width:800px;
            height: 250px;
            background-color: lightyellow;
        }
    </style>
</head>
                
<body>
    <h1>Story Editor</h1>
    Story Name: <input id=name_input></input>
    <section>
        <button onclick="loadStory()">Find and Edit It</button>
        <button onclick="saveStory()">Save It</button>
        <button onclick="displayStory()">Display It</button>
    </section>
    <textarea id="story_editor" placeholder="Write a totally awesome story here."></textarea>
    <section id="story_display"></section>
</body>
</html>
Master Mobile UX

How to Rewire Your Brain for Mobile First UX

"If you design for the smallest, least powerfull device first, then you will focus on your conntent, and your core functionality. That leads to simple, beautiful apps. If you do it the other way around, it'll be like trying to put a marshmallow into a piggy bank, which is either simple, nor beautiful." Joel Marsh - The Hipper Element
  • More and more consumers have a mobile device. Everything we build, the application needs to be viewed accross all devices.
  • We have been making websites and applications for larger screens.
  • Consumers will have 3-5 devices, meaning multiple devices per houshold
"The simple guideline is whatever you are doing - do mobile first" - Eric Schmidt, World Mobile Congress Keynote 2012
  • Responsive != Mobile First
  • Responsive Retrofitting
    • Cheaper and faster than a group rewrite
    • Better mobile experience than a desktop only site
    • Doesn't address screen real estate issues
    • Doesn't address touch interaction issues
  • Mobile First
    • Time consuming and costly for a full rebuild
    • Proviedes best experience accross many devices
    • Address constraints of mobile size/bandwidth
    • Future friendly and ready for growth
  • Things to Remember
    • Desktop computer sales are way down, mobile devices are in much more demand.
    • Start small and scale up.
    • Mobile is not going away.
  • Build Mobile first!!
  • Expand from the mobile, it will make it easier
  • Try to simplify the process
  • Minify CSS and JS files
  • Make forms show only the data that they are willing to provide ie email or phone number based on checkboxes
  • Use number pad for certain fields
  • Use anchor tags to be able to have users click to call.
  • Stop using pop-ups
  • Make it sleek and focus on content first

How to Improve UX by Understanding Users

"It's really hard to design products by focus groups. A lot of times, people don't know what they want until you show it to them." Steve Jobs - Apple
  • Be There!
  • Be Useful! Only 9% of users will stay if it doesn't satisfy their needs
  • Be Quick! It's not about fast loading, the functionality must be there as well.
  • Reachability matters
  • Speed Matters
  • Networks Matter
  • Reduce image usage throught sites/apps
  • SVG/Webfonts are lower bandwidth
  • Optimize and Minify CSS and JS files
  • GZIP files to eliminate unnecessary data

Designing Intuitive and Mobile Friendly Form

"[The constraints of Mobile] push you toward more focused, simplifies solutions. You can only fit so much on the screen, people often have to wait for it, and they're unlikely to give you their full attention. So make it easy to understand and use focus on the important things first. Mobile is a great forcing function for simplicity." Luke Wroblewski - Co-Founder of Polar and Bagcheck
  • The big problem with forms these days is the ammount of information required of the user.
  • Have input lables displayed in the input field.
  • Use float lables.
  • Get rid of unnecessary fields.
  • If you need a form with more information, use pagination. Also include a progress bar so the user knows how long it will be.
  • Use Autocomplete on your forms.
"Typically, masking passwords doesn't even increae security, but it does cost you business due to login failures." - Nielsen Norman Group
  • Auto advancing when entering credit card information helps the user reduce errors.
  • Provide users the best keyboard for the information they are entering.
  • Make phone numbers calable from your website.
